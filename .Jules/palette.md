## 2024-05-14 - Interactive Div Cards Need Keyboard Support and Event Isolation
**Learning:** This app frequently uses clickable `div` elements for navigation (like `ProductCard`), which breaks keyboard accessibility since `div`s aren't naturally focusable or interactive. Furthermore, when interactive elements (like a "Add to Wishlist" or "Buy Now" button) are nested inside these clickable cards, clicking the nested button triggers the parent's navigation due to event bubbling.
**Action:** Always add `role="link"` or `role="button"`, `tabIndex={0}`, `onKeyDown` handlers (for Enter/Space), and visible focus states (`focus-visible:ring-2`) to interactive `div`s. For nested interactive elements, ensure they have `onClick={(e) => e.stopPropagation()}` to prevent unintended parent actions, and give them their own focus states.
