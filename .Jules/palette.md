## 2024-05-24 - Accessibility improvements to ProductCard.jsx
**Learning:** Found an accessibility issue pattern specific to this app's components, where nested interactive elements do not stop propagation, causing unintended side effects (e.g., clicking wishlist navigates to the item details page), and repetitive decorative images not having empty `alt` attributes causing repetitive announcements by screen readers.
**Action:** Use `onClick={(e) => e.stopPropagation()}` on interactive elements nested within clickable container components. Set `alt=""` to decorative images to prevent screen readers from redundantly announcing repetitive elements, and `aria-hidden="true"` to groups of purely visual representations of data that are presented as text elsewhere in the same context.
