## 2024-04-11 - Nested Interactive Elements in ProductCard
**Learning:** Using a clickable `div` wrapper for navigation (like in `ProductCard`) is common but causes significant accessibility and interaction issues when nested with buttons (like wishlist or buy). Without event propagation stopping, interacting with nested buttons incorrectly triggers the parent container's routing logic. Additionally, `div` wrappers should not use `role="button"` if they contain interactive elements to adhere to WCAG standards.
**Action:** Always use `onClick={(e) => e.stopPropagation()}` on all interactive elements nested inside clickable containers to prevent unintended navigation. Ensure focus states are manually managed (e.g., `focus-visible:ring-2`) and aria attributes correctly guide the screen reader context on these elements.
