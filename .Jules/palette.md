## 2024-05-24 - Interactive Elements inside Clickable Cards
**Learning:** When dealing with UI components acting as clickable cards (e.g., `ProductCard` wraps its content in a `div` with `onClick` to navigate to a product details page), placing interactive elements like buttons (wishlist, "Buy now") inside these cards creates usability and accessibility issues. If users try to click the button, the click event bubbles up to the parent card, triggering the card's action (navigation) instead of or in addition to the button's intended action. This is particularly problematic for screen reader and keyboard users who might trigger actions unintentionally. Nesting interactive elements like buttons inside other interactive elements is a severe WCAG accessibility violation. The outer element should ideally not be a semantic structural tag like `div` acting as a button without the `button` role, but if it is, `role="button"` shouldn't be added to the outer `div` because of nested interactive elements.
**Action:** Always use `onClick={(e) => e.stopPropagation()}` on interactive elements nested within clickable container components. This ensures the nested action triggers independently and prevents the parent container's click events from being unintendedly triggered.

## 2024-05-24 - Redundant Visual Ratings
**Learning:** Visual star ratings (using images or icons) paired with a text-based rating (e.g., "4.5") cause redundant and confusing announcements for screen reader users if both are exposed.
**Action:** Hide the image container for visual star ratings using `aria-hidden="true"` and give individual star images an empty `alt` attribute (`alt=""`). Ensure the accompanying text rating provides the full context, optionally using `aria-label` for better screen reader formatting (e.g., `aria-label="Rating: 4.5 out of 5 stars"`).
