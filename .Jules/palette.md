## 2024-04-08 - ProductCard Component Accessibility Patterns
**Learning:** In Next.js App Router applications, improving the accessibility of interactive sub-elements (like wishlist buttons) within larger clickable container components (like Product Cards) requires converting the component to a Client Component (`"use client"`) to safely utilize `e.stopPropagation()`. Additionally, purely decorative visual star ratings paired with numerical text ratings must be hidden from screen readers using `aria-hidden="true"` on the container and empty `alt=""` text on individual images to prevent severe redundancy in screen reader announcements.
**Action:** Always verify if a container element is clickable before adding inner interactive elements. If it is, convert the file to a Client Component and apply `e.stopPropagation()`. When visual ratings accompany text ratings, strictly apply `aria-hidden="true"` to the visual container.
