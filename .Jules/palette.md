## 2024-03-15 - Redundant Screen Reader Announcements and Nested Interactivity
**Learning:** Visual star ratings that are paired with text ratings (like "4.5") cause screen readers to redundantly announce "star icon" multiple times if not hidden. Also, placing interactive elements (like a wishlist button) inside a clickable card `div` creates usability issues if event propagation is not stopped, potentially triggering both actions simultaneously.
**Action:** Always wrap visual repeating icon components (like star ratings) in a container with `aria-hidden="true"` and use empty `alt=""` attributes on the icons when a text equivalent is present. Always ensure nested interactive elements use `e.stopPropagation()` in their click handlers to prevent unintended activation of parent click events.
