## 2024-03-24 - ProductCard Accessible Clickable Container
**Learning:** When using a `div` as a clickable container (like `ProductCard.jsx`) for navigation, it must be properly treated as a link/button using `role="link"`, `tabIndex={0}`, and an `onKeyDown` handler to support keyboard navigation. Furthermore, nested interactive elements (like wish-list buttons or "Buy Now" buttons) *must* implement `e.stopPropagation()` on their click events. Otherwise, interacting with the nested button will trigger the outer div's navigation, confusing the user and breaking the intended component action. Icon-only buttons also require `aria-label`s.
**Action:** When auditing or building card components that navigate when clicked, always verify keyboard navigability and ensure all nested buttons stop event propagation. Ensure `focus-visible` styling is present for clear focus indicators.
