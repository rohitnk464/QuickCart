## 2026-03-22 - Accessibility for Nested Interactive Elements and Visual Ratings
**Learning:** Adding interactive elements (like buttons) inside clickable container elements creates an accessibility issue if not handled properly. The nested element click event triggers the parent container's event, leading to unintended behavior (e.g. clicking the wishlist button navigating the user to the product page). Redundant visual star ratings accompanying text ratings also clutter screen reader outputs.
**Action:** Always use `onClick={(e) => e.stopPropagation()}` on child interactive elements when they are placed inside a clickable wrapper to prevent event bubbling. Additionally, add `aria-hidden="true"` to visual rating containers and use empty `alt=""` for the individual icons when the numerical rating is already provided in text form.
