## 2024-05-24 - Interactive Container Keyboard Accessibility & Event Propagation
**Learning:** Found a common pattern where a `<div onClick={...}>` is used as a full-card clickable area (e.g., `ProductCard.jsx`), but lacking keyboard support. Furthermore, nested interactive elements (like "Wishlist" or "Buy now" buttons) inside such containers triggered both their own actions AND the parent card's navigation due to event bubbling. This causes unintended page transitions for users interacting with specific buttons.
**Action:** When creating or modifying clickable container elements (`div`, `article`, etc.), always ensure they have `role="link"` or `role="button"`, `tabIndex={0}`, and an `onKeyDown` handler (for Enter/Space).  Crucially, any nested interactive elements (buttons, links) within these containers *must* use `onClick={(e) => e.stopPropagation()}` to prevent event bubbling and accidental triggering of the parent container's action. Additionally, ensure clear focus indicators (`focus-visible:ring-2`, etc.) are applied to all interactive targets.
