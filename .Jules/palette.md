## 2024-03-24 - Nested Interactive Elements and Redundant Visuals in ProductCards
**Learning:** In complex UI components like `ProductCard`, interactive elements (like a wishlist button or a 'buy now' button) nested within a clickable container element (like a card that navigates to a detail page) cause severe UX issues if their events propagate. The outer navigation is unintentionally triggered when trying to interact with the inner buttons. Furthermore, adding text-based ratings next to visual star ratings causes screen readers to redundantly announce 'star_icon' multiple times if the star images have their alt attributes set and aren't hidden via `aria-hidden`.
**Action:** Always use `onClick={(e) => e.stopPropagation()}` on interactive elements nested inside clickable containers. For visual representations of data that are accompanied by a text equivalent (like star ratings), ensure the container has `aria-hidden="true"` and the individual images have empty alt text (`alt=""`) to prevent redundant and annoying announcements.
